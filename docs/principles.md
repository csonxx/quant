# 量化交易原理：先把脑子调准

量化交易不是“用代码预测明天涨跌”。更准确地说，它是在不确定市场里，用可复现的数据和规则，反复寻找一个扣掉成本后仍然为正的期望值。

最核心的式子只有一个：

```text
期望收益 = 胜率 * 平均盈利 - 败率 * 平均亏损 - 交易成本 - 冲击成本
```

如果一个策略只会说“胜率高”，但平均亏损巨大，它可能是灾难。如果它只会说“收益高”，但换手高到成本吃掉一切，它也是灾难。如果它只在一段历史上有效，却解释不了为什么未来还有同样的买方或卖方行为，它只是历史拟合。

## 0. 小白先抓住一条主线

先别把量化想成“找公式预测价格”。更好的理解是：

```text
想法 -> 数据 -> 规则 -> 回测 -> 反证 -> 风控 -> 纸面交易 -> 复盘
```

每一步都在问一个朴素问题：

- 想法：我凭什么认为这里有优势？
- 数据：这些信息在当时真的能看到吗？
- 规则：别人能不能按同样规则做出同样结果？
- 回测：过去按这个规则做，账本怎么变化？
- 反证：哪些条件一改，它马上失效？
- 风控：错了最多亏多少？
- 纸面交易：不用真金白银时，它是否还能按预期运行？
- 复盘：亏损是正常波动、执行错误，还是策略死了？

这条链断在哪，策略就死在哪。小白阶段最重要的是把链条走完整，而不是急着找最复杂的模型。

## 1. Alpha、Beta 和噪声

市场收益可以粗暴拆成三块：

- `Beta`：你暴露在市场、行业、风格上的共同风险，比如大盘上涨带来的收益。
- `Alpha`：扣除这些共同风险后，策略规则自身带来的超额收益。
- `Noise`：随机波动、数据错误、偶然样本、运气。

新手最常犯的错，是把 Beta 和 Noise 当成 Alpha。比如牛市里随便买强势股都赚钱，然后误以为自己的选股模型有效。毒辣一点看：如果你的策略没有跑赢一个低成本指数或一个简单风格基准，你大概率没有策略，只有市场暴露。

## 2. 策略不是信号

一个交易信号只回答“方向可能怎样”。一个策略至少要回答：

- `Universe`：在哪些资产里交易，为什么是这些资产。
- `Signal`：什么条件触发买卖，信号在经济上为什么可能有效。
- `Position`：信号如何转成仓位，满仓、半仓、等权、波动率目标，还是风险预算。
- `Execution`：何时成交，用什么价格假设，手续费、滑点、停牌、涨跌停怎么处理。
- `Exit`：何时退出，是反向信号、止损、止盈、时间止损，还是组合再平衡。
- `Risk`：单笔最大亏损、组合回撤、相关性、流动性、容量上限。
- `Review`：失败后怎么归因，哪些条件触发永久下线。

所以移动均线交叉不是策略，它只是信号。把它放进仓位、成本和退出规则之后，才开始像策略。

## 3. 回测是什么，不是什么

回测是历史模拟器，不是赚钱证明。它能帮你回答“如果当时按这套规则做，大概会发生什么”。它不能证明未来一定发生同样结果。

一个可信回测必须至少处理这些问题：

- `Look-ahead bias`：用了当时还不知道的数据。最常见是用当天收盘价生成当天收盘成交。
- `Survivorship bias`：只看今天还活着的股票，忽略退市和失败样本。
- `Selection bias`：看了很多策略，只展示最漂亮的一条。
- `Data snooping`：反复调参直到历史最优，但未来崩掉。
- `Cost blindness`：不算手续费、滑点、冲击、借券成本、税费。
- `Liquidity fantasy`：小票上假设无限成交，收益很好看但现实买不进去也卖不出。
- `Corporate actions`：分红、拆股、复权处理不一致。

最危险的回测不是收益差的回测，而是收益漂亮、逻辑含糊、成本很低、参数很多、样本很短的回测。它会让人产生“我发现了规律”的幻觉。

## 4. 为什么过拟合这么容易

金融数据的信噪比很低。你只要尝试足够多的参数、因子、过滤条件，总能找到一条历史上很好看的曲线。

这不是你聪明，是搜索空间太大。假设你测试 500 个策略配置，即使每个配置都没有真实 alpha，其中也很可能出现几个夏普很高的幸运儿。问题在于你最后只记住了幸运儿，忘了那 499 个坟场。

所以本仓库要求保留：

- 尝试了哪些策略，不只保留成功的。
- 每次参数范围是什么。
- 样本内和样本外怎么切。
- 和基准相比是否真的有增益。
- 成本、换手、容量是否杀死收益。

## 5. 样本外、走步和 Purged Split

时间序列不能像普通表格数据那样随便随机切分。因为今天的标签可能依赖未来几天收益，训练样本和测试样本之间可能共享信息。

更稳的验证顺序：

1. `Train`：只用早期历史开发信号。
2. `Validation`：用中间时期选择少量参数。
3. `Test`：最后一段时间只看一次，不能再调。
4. `Walk-forward`：滚动训练、滚动测试，检查策略是否稳定。
5. `Purged/Embargoed split`：当标签窗口重叠时，清除训练和测试之间的信息泄漏。

毒辣判断：如果一个策略只有一次切分有效，它还不是策略，只是候选。稳定不是“每年都赚”，而是亏损年份也能解释，收益来源没有彻底失效。

## 6. 成本和滑点为什么有一票否决权

很多策略的毛收益来自微小价差。只要换手高，手续费和滑点会把它吃干净。

你要把成本分成几类：

- 固定费用：佣金、平台费、最低收费。
- 比例费用：按成交金额收取。
- 买卖价差：用中间价回测太乐观。
- 市场冲击：你的订单越大，越会把价格推向不利方向。
- 机会成本：挂单没成交，或成交太慢。

如果一个策略毛年化 12%，年换手 800%，单边成本 5bp，光比例成本就可能接近 `800% * 2 * 0.05% = 8%`。再加滑点和冲击，剩下的肉可能只有骨头味。

## 7. 风险不是波动率一个数字

风险至少包括：

- `Drawdown`：从高点跌到低点，决定你能不能心理和资金上扛住。
- `Tail risk`：小概率大亏，均值和夏普可能看不出来。
- `Correlation`：看似分散的仓位，压力时可能一起跌。
- `Crowding`：太多人做同一笔交易，退出时踩踏。
- `Regime shift`：市场结构变了，过去有效的行为消失。
- `Operational risk`：数据延迟、代码 bug、订单错误、停牌、限价。

一个策略如果只展示年化和夏普，不展示最大回撤、最长回撤恢复期、月度分布、单笔亏损、极端时期表现，就是在藏刀。

## 8. 因子研究的正确姿势

因子不是“看起来有道理的指标”。因子要过几关：

1. 有经济含义：为什么这个行为会被市场长期奖励。
2. 可观测：数据当时真的可获得，发布时间不能穿越。
3. 可交易：换手、流动性、容量能承受。
4. 可分层：高分组和低分组表现有稳定差异。
5. 可衰减：信号不会一公布就消失，或至少知道半衰期。
6. 可组合：和已有因子不是同一个东西的马甲。

横截面因子尤其要小心行业、市值、中性化和极端值处理。很多“强因子”其实只是小市值暴露，或者某一年某个行业大涨的残影。

## 9. 机器学习在量化里真正难在哪里

难点不是训练模型，而是定义一个不泄漏、不自欺、可交易的预测问题。

你必须先定义：

- 预测对象：未来 1 天、5 天、20 天收益？还是超额收益？还是涨跌分类？
- 标签时间：标签依赖未来多久，训练切分要不要 purge。
- 特征时间：特征在交易时点是否已经发布。
- 目标函数：预测准确率、排序能力、组合收益，还是风险调整收益。
- 交易翻译：模型输出如何变成仓位。

机器学习很擅长在脏数据里找到历史巧合。它会把你的偏差放大，不会自动变聪明。没有严格验证，ML 只是高级曲线拟合器。

## 10. 工具选择

先手写最小引擎，是为了理解账本、时序和成本。理解之后再上工具：

- `vectorbt`：适合大规模向量化参数扫描和 notebook 研究。
- `Qlib`：适合 AI/因子/模型研究流水线。
- `LEAN/QuantConnect`：适合事件驱动、多资产、接近实盘的算法框架。
- `backtrader`：适合学习事件驱动回测和策略对象模型。

不要一开始就被大框架吞掉。框架能加速你，也能替你隐藏错误。先知道每一笔权益曲线是怎么算出来的，再谈加速。

## 11. 本仓库的训练方法

每个模块都按这个闭环推进：

```text
原理 -> 手算小样本 -> 写最小代码 -> 加测试 -> 跑样例 -> 写反例 -> 再扩展
```

如果一个概念不能在 20 行数据上手算明白，就不要急着把它扔进 5000 只股票和 200 个因子里。规模会放大你的混乱。

## 12. 交易审判的最低标准

一个候选策略要进入下一阶段，至少回答：

- 它赚的是谁的钱？对手方为什么长期愿意输？
- 如果反过来做会怎样？
- 去掉最赚钱的 5 个交易后还赚钱吗？
- 加 2 倍成本还赚钱吗？
- 延迟一根 K 线成交还赚钱吗？
- 换一个市场、行业、年份还活着吗？
- 它最怕什么市场状态？
- 如果连续亏损 10 次，是否还能执行？

回答不了，就先放进墓地。量化研究不是证明自己对，而是高效淘汰自己错。
