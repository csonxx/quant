# 研究协议

每个策略研究都必须留下证据链。目标是让三个月后的自己能复现，也能看懂当时为什么通过或失败。

## 研究记录字段

- `idea_id`：唯一编号，例如 `MOM-001`。
- `hypothesis`：策略假设，用一句话讲清楚。
- `economic_reason`：为什么这个现象可能长期存在。
- `universe`：资产范围、过滤条件、是否存在幸存者偏差。
- `data_sources`：数据来源、字段、时间戳、复权方式。
- `signal`：信号计算公式，不允许只写“模型预测”。
- `positioning`：仓位规则，含杠杆、做空、再平衡频率。
- `execution_assumption`：成交价格、费用、滑点、停牌、涨跌停。
- `baseline`：至少一个朴素基准。
- `validation`：样本内、样本外、走步或 purged split。
- `results`：收益、回撤、夏普、换手、容量、失败时期。
- `counter_evidence`：最强反例。
- `decision`：`kill`、`observe`、`paper_trade`、`promote`。

## 禁止事项

- 禁止只保存成功回测图。
- 禁止回测后再补写假设。
- 禁止用测试集调参。
- 禁止不算成本就讨论收益。
- 禁止把模型分数直接叫交易机会。
- 禁止没有退出规则的买入信号。

## 最低验收

一个研究要从 `idea` 进入 `candidate`，至少满足：

1. 代码可复现，`make test` 通过。
2. 有基准对照。
3. 有费用和滑点假设。
4. 有样本外结果。
5. 有失败场景说明。
6. 有明确的下线条件。
