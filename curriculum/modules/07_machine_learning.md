# 07 机器学习

## 为什么最后才学

机器学习不是量化的捷径。它能放大优势，也能放大错误。

如果你还没学会数据契约、前视偏差、回测成本、样本外验证，机器学习只会更快地制造漂亮幻觉。

## 核心原理

### 1. 先定义预测问题

模型不能回答“买什么赚钱”这种模糊问题。你必须明确：

- 预测未来 1 天、5 天还是 20 天？
- 预测绝对收益还是超额收益？
- 做回归、分类还是排序？
- 预测结果如何变成仓位？

问题定义错了，模型再强也没用。

### 2. 标签

标签是模型要学习的答案。

例子：

```text
5 日未来收益 = close[t+5] / close[t] - 1
```

如果你在 t 日训练特征，标签用 t+5 是可以的；但切分训练和测试时要注意，窗口会重叠，可能泄漏。

### 3. 特征

特征是模型在预测时能看到的信息。

特征必须满足：

- 在预测时点已经发布。
- 不包含未来价格。
- 不因为数据回填而穿越。
- 和标签窗口不混淆。

### 4. 时间序列切分

普通随机交叉验证在金融时间序列里经常有毒，因为未来样本可能混进训练。

更稳的方式：

```text
早期训练 -> 中期验证 -> 后期测试
滚动训练 -> 滚动测试
Purged split -> 清理标签窗口重叠
Embargo -> 测试集附近留空
```

### 5. 预测指标不等于交易收益

模型准确率高，不代表交易赚钱。

原因：

- 预测对的小涨，预测错的大跌。
- 排名有效但换手太高。
- 模型只学到市场 beta。
- 成本吃掉小优势。

量化 ML 最终要回到组合收益、回撤、换手、容量。

## 手算例子

今天 t，特征用过去 5 天收益，标签用未来 5 天收益。

如果训练集结束在第 100 天，测试集从第 101 天开始：

```text
第 100 天样本的标签需要第 105 天价格
第 101-105 天已经进入测试窗口
```

这就有信息重叠风险。需要 purge 掉边界附近样本。

## 代码练习

先不要训练模型。先画窗口：

```text
feature window: t-5 ... t
label window:   t ... t+5
```

练习目标：

- 给每个样本标注特征起止日期。
- 给每个样本标注标签起止日期。
- 找出训练/测试边界附近需要 purge 的样本。

## 常见误区

- 误区：准确率高就是好模型。
  真相：要看交易收益和风险。
- 误区：随机切分样本更多、更科学。
  真相：时间序列会泄漏未来。
- 误区：特征越多越好。
  真相：特征越多越容易过拟合。
- 误区：模型能自动发现 alpha。
  真相：模型更擅长发现你数据里的偏差。

## 毒辣验收

你必须能回答：

1. 标签窗口和特征窗口分别是什么？
2. 为什么随机交叉验证可能泄漏？
3. 为什么准确率高也可能亏钱？
4. 为什么 ML 输出必须经过仓位和风控？
5. 什么情况下宁愿不用模型？
