# 02 数据契约和偏差

## 为什么学这个

量化里最贵的错误通常不是公式写错，而是数据在骗人。数据错了，模型越复杂，错得越漂亮。

数据契约就是先规定：这份数据必须有哪些字段、字段是什么意思、什么时候可知、能不能用于当时交易。

## 核心原理

### 1. OHLCV

最基础的行情数据：

- `open`：开盘价。
- `high`：最高价。
- `low`：最低价。
- `close`：收盘价。
- `volume`：成交量。

本仓库的样例数据在：

```text
data/samples/ohlcv_demo.csv
```

### 2. 时间戳是第一等公民

同一个数值，在不同发布时间下意义完全不同。

例子：

- 今天 15:00 后才知道收盘价。
- 财报可能晚上发布。
- 指数成分调整可能提前公告，也可能在生效日才进入数据库。

如果你的策略在上午 10:00 用了下午 15:00 才知道的数据，这就是穿越。

### 3. 前视偏差

前视偏差是用了当时不知道的信息。

典型错误：

```text
用今天收盘价计算信号，然后假装今天收盘价成交。
```

正确思路：

```text
今天收盘生成信号，最快明天成交。
```

### 4. 幸存者偏差

如果你的股票池只包含今天还活着的股票，就漏掉了退市、长期停牌、被并购或失败的公司。

结果会虚高，因为失败者被删掉了。

### 5. 复权偏差

分红、拆股、送转会改变价格序列。如果复权方式不一致，收益计算会错。

你需要明确：

- 用前复权还是后复权。
- 成交价是否也用复权价。
- 股息是否计入收益。

### 6. 字段检查

最小检查：

- 必填字段是否存在。
- 日期是否递增。
- 价格是否为正。
- `low <= open/close <= high` 是否成立。
- 成交量是否非负。
- 是否有重复日期。

## 手算例子

如果某天数据是：

```text
open=100, high=98, low=95, close=97
```

这不可能，因为最高价比开盘价还低。加载器应该拒绝。

如果某个策略在 `2024-01-05` 使用了 `2024-01-08` 的收盘价算信号，这不是聪明，是作弊。

## 常见误区

- 误区：数据能下载就能用。
  真相：能下载只是开始，先问字段含义和发布时间。
- 误区：缺失值填 0 很方便。
  真相：价格填 0 会制造巨大假收益。
- 误区：复权价格和真实成交价可以混用。
  真相：这会让回测成交账混乱。
- 误区：股票池用当前成分股就行。
  真相：这会引入幸存者偏差。
