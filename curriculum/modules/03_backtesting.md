# 03 回测引擎

## 为什么学这个

回测是量化学习里最会骗人的部分。它能让一个烂想法看起来像提款机。

学回测不是为了相信回测，而是为了知道它在哪里可能作弊。

## 核心原理

### 1. 回测账本

最小回测账本包括：

```text
日期
价格
信号
目标仓位
成交成本
当日收益
账户权益
```

如果你不能逐行解释权益曲线怎么变出来，就不要相信最终收益。

### 2. 信号和仓位

信号是判断，仓位是行动。

例子：

```text
信号：快均线 > 慢均线，看多
仓位：账户 100% 买入
```

同一个信号可以对应不同仓位：

- 满仓。
- 半仓。
- 波动率越高仓位越低。
- 最大亏损固定为账户 1%。

所以信号不是策略，仓位规则才把信号变成交易。

### 3. 成交时序

本仓库当前回测规则：

```text
t-1 日信号 -> 控制 t-1 到 t 的收益暴露
```

也就是上一根 K 线信号决定下一段收益，避免用当天收盘信息偷看当天收盘成交。

### 4. 成本

最小成本模型：

```text
费用 = 当前权益 * 仓位变化绝对值 * fee_bps / 10000
```

如果从空仓到满仓，仓位变化是 1。若权益 10000，费用 5bp：

```text
10000 * 1 * 5 / 10000 = 5
```

如果从满仓到空仓，又付一次。

### 5. 买入持有基准

任何单资产策略都应该先和买入持有比。

如果策略折腾半天，收益更低、回撤更大、成本更高，那就别包装了，直接判弱。

## 手算例子

价格：

```text
100 -> 110 -> 105
```

信号：

```text
第 1 天结束后满仓
第 2 天结束后继续满仓
```

初始资金 10000，费用 0。

账：

```text
第 1 天：还没持仓，权益 10000
第 2 天：满仓吃到 100 -> 110，权益 11000
第 3 天：满仓吃到 110 -> 105，权益 10500
```

如果你用第 2 天收盘价 110 生成第 2 天信号，又让它吃到 100 -> 110 的收益，那就是作弊。

## 常见误区

- 误区：回测收益曲线越平滑越好。
  真相：可能是偷看未来或成本太乐观。
- 误区：参数调到最好就是最好策略。
  真相：大概率是过拟合。
- 误区：只要夏普高就能交易。
  真相：要看样本外、回撤、容量、换手、成本。
- 误区：费用很小可以先忽略。
  真相：高换手策略常常死在费用上。
