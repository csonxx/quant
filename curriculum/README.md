# 学习路线

每个模块都产出三样东西：

- 一份能讲清原理的笔记。
- 一个能跑的小实验。
- 一个能杀死错误结论的测试或反例。

## 总判断

量化学习最容易走错两条路：

- 先学一堆指标，以为指标组合就是策略。
- 先上大框架和机器学习，以为工具复杂就更接近赚钱。

这个仓库反过来走：先用小数据、小代码、小测试，把每个收益数字怎么来的讲明白。等你能解释每一笔权益变化，再扩大到多资产、因子、组合和机器学习。

## 四阶段路线

| 阶段 | 模块 | 目标 | 不能跳过的原因 |
| --- | --- | --- | --- |
| 0 小白地基 | 00 | 建立正确心智模型，补齐最小数学和 Python 数据能力 | 不懂期望值和数据表，后面所有回测都会变成玄学 |
| 1 单资产闭环 | 01-04 | 用一个标的讲清价格、收益、数据、回测、风控 | 单资产账本不清楚，多资产只会放大混乱 |
| 2 多资产研究 | 05-06 | 学因子、组合、相关性、换手和容量 | 选股信号不等于组合收益 |
| 3 模型和执行 | 07-08 | 学 ML、纸面交易、监控、复盘、下线 | 模型输出不是交易结论，实盘风险来自执行和失控 |

## 模块表

| 模块 | 主题 | 交付物 | 通过标准 |
| --- | --- | --- | --- |
| 00 | 小白预备课 | 能讲清量化到底在赌什么、用什么证伪 | 能用期望值解释为什么高胜率也会亏钱 |
| 01 | 市场、收益和基准 | 手算收益和最大回撤 | 能解释收益、波动、回撤、基准差异 |
| 02 | 数据契约和偏差 | OHLCV 加载器和数据检查 | 能指出前视、幸存者、复权问题 |
| 03 | 回测引擎 | 最小信号回测 | 信号延迟、费用、权益曲线测试通过 |
| 04 | 风险和仓位 | 交易计划审查器 | 能把信号转成仓位和亏损上限 |
| 05 | 因子研究 | 横截面因子实验 | 有 IC、分组、换手、样本外 |
| 06 | 组合构建 | 再平衡和风险预算 | 能解释相关性和集中度 |
| 07 | 机器学习 | 标签/特征/切分实验 | 无泄漏，有 purged/走步验证 |

学习顺序不要跳。你可以先读后面的，但代码深化要按顺序来，因为后面的每个结论都依赖前面的数据和回测纪律。

## 阶段门禁

### 阶段 0 通过标准

- 能解释 `期望收益 = 胜率 * 平均赚 - 败率 * 平均亏 - 成本`。
- 能读懂一张 OHLCV 表。
- 能写一个 Python 循环计算收益。
- 能接受“暂时没有交易机会”是正常结论。

### 阶段 1 通过标准

- 能从价格手算收益、权益曲线和最大回撤。
- 能解释为什么今天收盘生成的信号不能假装今天收盘成交。
- 能跑 `make demo`，并解释每个指标不是魔法数字。
- 能把一个买入信号翻译成仓位、止损、目标价和最大亏损。

### 阶段 2 通过标准

- 能解释横截面排序和时间序列择时的区别。
- 能说清因子分组、IC、换手、容量分别杀什么假策略。
- 能解释为什么相关性会上升，为什么组合会一起跌。

### 阶段 3 通过标准

- 能画出特征窗口和标签窗口，指出泄漏风险。
- 能用 walk-forward 或 purged split 做验证。
- 能写纸面交易规则和下线条件。
- 能把一次亏损分成策略内亏损、执行问题、数据问题或策略失效。
